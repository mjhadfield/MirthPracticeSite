Explanation

This page is for me to try and get more familiar with MirthConnect. The forms on this site use NodeJS to send the information into a local MSSQL database. This is purely so I can get a few tables going, and use database readers in MirthConnect to try and do some stuff.

The new patient form will generate a new entry in the "newpatients" table and assign them a unique hospital number. Mirth will then process that row, generate an ADT message and update the request demographics table with patient information. Similarly, the order form will send a request to an orders table, which Mirth then pulls to generate an OBR.

Next up I want to try and fudge some "analyser" interfacing, send those OBRs off to another interface which pretends to run it, (somehow) generate results, and then post the results back to the database.

I don't know, just seeing what happens really.

Oh and if you're looking at the source code on GitHub, most of it is disgusting. The vast majority of the JavaScript, HTML, and CSS is AI generated, but I don't know what you expected.
